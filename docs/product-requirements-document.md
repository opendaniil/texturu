# PRD TubeBook v0.8 — YouTube Article + RAG Chat
---

## 1. Цели продукта

### Бизнес-цель

Публичное приложение для портфолио, демонстрирующее навыки full-stack разработки с интеграцией AI и фоновой обработки.

### Пользовательская цель

Помочь пользователю быстро понять содержание длинного YouTube-видео без полного просмотра и задать уточняющие вопросы по содержанию.

---

## 2. Целевая аудитория

### Персоны

Студент / исследователь

- Контекст: изучает тему по лекциям и подкастам
- Боль: нет времени смотреть 2-часовое видео целиком
- Задача: быстро получить структурированное саммари, найти нужный момент

Профессионал

- Контекст: следит за индустрией через YouTube-интервью
- Боль: сложно вернуться к конкретной мысли спикера
- Задача: задать вопрос и получить ответ с указанием источника

Админ (владелец)

- Контекст: поддерживает работоспособность проекта
- Боль: нужно понимать, что сломалось
- Задача: видеть статусы задач, перезапускать обработку

---

## 3. Ценностное предложение

Экономия времени

- Реализация: статья вместо просмотра — 5 минут чтения vs 60+ минут видео

Доверие к ответам

- Реализация: каждый ответ чата подкреплён цитатами из видео с таймкодами

Прозрачность

- Реализация: пользователь всегда понимает, на каком этапе обработка и почему что-то не сработало

Непрерывность диалога

- Реализация: чат продолжает историю на том же устройстве между перезагрузками/вкладками за счёт HTTP-only cookie

---

## 4. Метрики успеха (MVP)

Конверсия URL → готовая статья: >80% (для видео с субтитрами)

- Как измерять: соотношение статусов в базе данных

Среднее время обработки: <3 минут для видео до 60 минут

- Как измерять: разница timestamps начала и окончания задачи в логах

Использование чата: >30% посетителей страницы видео отправляют хотя бы 1 вопрос

- Как измерять: счётчик отправленных сообщений в логах backend

Ответы с цитатами: >90% ответов содержат хотя бы 1 цитату

- Как измерять: логи ответов чата (наличие поля citations)

---

## 5. Область и ограничения (MVP)

Вход: URL одного YouTube-видео.
Выход: структурированная статья + чат с цитатами.

### Ограничения MVP и поведение системы

Видео без субтитров

- Поведение: сообщение «Для этого видео нет субтитров. Попробуйте другое видео.»

Недоступное видео (удалено, приватно, geo-блокировка)

- Поведение: сообщение «Видео недоступно. Проверьте ссылку или попробуйте другое видео.»

Ошибка YouTube API (rate limit, сетевая ошибка)

- Поведение: сообщение «Не удалось загрузить информацию о видео. Попробуйте позже.»

Длительность видео

- Лимит: до 120 минут
- Для более длинных: сообщение «Видео слишком длинное. Максимальная длительность — 2 часа.»

Язык

- Субтитры: принимаются на любом языке
- UI, статья, ответы чата: всегда на русском языке

Таймкоды в цитатах

- Показываем временной диапазон, но без кликабельной перемотки в MVP

Хранение данных

- Видео и статьи хранятся бессрочно

Параллельная обработка

- Задачи обрабатываются через очередь с настраиваемым количеством воркеров

## 7. Пользовательские сценарии

### Сценарий 1: Добавление нового видео (Happy Path)

1. Пользователь на главной видит поле ввода URL и список видео
2. Вставляет YouTube URL и нажимает «Добавить»
3. Происходит редирект на страницу видео
4. Пользователь видит встроенный плеер, название, статус «В очереди»
5. Статус обновляется: «Получаем субтитры» → «Обрабатываем» → «Готово»
6. Появляются статья и чат

Ожидание пользователя: обработка занимает 1–3 минуты. Страница обновляется автоматически (polling каждые 5 секунд).

### Сценарий 2: Видео без субтитров

1. Пользователь добавляет URL
2. Редирект на страницу видео
3. Статус меняется: «В очереди» → «Получаем субтитры» → «Нет субтитров»
4. Пользователь видит:
   - Статус: «Нет субтитров»
   - Пояснение: «Это видео не содержит субтитров, поэтому мы не можем создать статью. Попробуйте другое видео.»

### Сценарий 3: Повторное добавление того же URL

1. Пользователь вставляет URL видео, которое уже есть
2. Редирект на существующую страницу видео (идемпотентность)
3. Пользователь видит текущий статус (готово / в обработке / ошибка)

### Сценарий 4: Чат с вопросом (обновлено: персистентный тред по cookie)

1. На странице готового видео пользователь открывает чат
2. Вводит вопрос: «Что автор говорит про X?»
3. Получает ответ с 1–3 цитатами
4. Каждая цитата показывает фрагмент текста и временной диапазон (например, «12:30–13:15»)
5. Пользователь обновляет страницу или открывает видео в новой вкладке на том же устройстве — история чата сохраняется и отображается

Если ответ не получен (ошибка сети, таймаут):

- Сообщение: «Не удалось получить ответ. Попробуйте ещё раз.»
- Кнопка «Повторить» рядом с неотправленным вопросом

### Сценарий 5: Ошибка обработки

1. Обработка завершается с ошибкой
2. Пользователь видит:
   - Статус: «Ошибка»
   - Сообщение: «Не удалось обработать видео. Попробуйте позже или выберите другое видео.»
3. Только для админа: кнопка «Перезапустить обработку»

### Сценарий 6: Админ — диагностика и перезапуск

1. Админ заходит в `/admin/jobs`
2. Видит список задач с фильтрацией по статусу/дате
3. Находит проблемную задачу и видит текст ошибки
4. Переходит на страницу видео и нажимает «Перезапустить»

### Сценарий 7: Возврат к видео в процессе обработки

1. Пользователь добавил видео, закрыл вкладку
2. Позже переходит на страницу видео (из списка или по прямой ссылке)
3. Видит актуальный статус обработки
4. Если готово — статья и чат доступны
5. Если в процессе — polling продолжается автоматически

### Сценарий 8: Недоступное видео

1. Пользователь добавляет URL удалённого или приватного видео
2. Редирект на страницу видео
3. Статус: «Ошибка»
4. Сообщение: «Видео недоступно. Проверьте ссылку или попробуйте другое видео.»

## 8. Функциональные требования

### 8.1 Добавление видео

- Валидация: только YouTube URL (youtube.com, youtu.be)
- Если URL уже добавлен: возврат существующего видео (идемпотентность)
- После создания: редирект на страницу видео
- Rate limit: не более 5 добавлений в минуту с одного IP

### 8.2 Статусы обработки

Список статусов и отображение для пользователя:

В очереди (промежуточный)

- Текст: «Видео в очереди на обработку»

Получаем субтитры (промежуточный)

- Текст: «Загружаем субтитры...»

Нет субтитров (финальный)

- Текст: «Субтитры недоступны. Обработка невозможна.»

Обрабатываем (промежуточный)

- Текст: «Создаём статью... Обычно это занимает 1–3 минуты»

Готово (финальный)

- Результат: статья и чат доступны

Ошибка (финальный)

- Текст: «Произошла ошибка. Попробуйте позже.»

### 8.3 Страница видео

Всегда отображает:

- Встроенный YouTube-плеер
- Название и канал
- Текущий статус с пояснением

При статусе «Готово» дополнительно:

- Статья
- Панель чата

### 8.4 Структура статьи

Разделы статьи:

- Заголовок: название видео
- О видео: канал, длительность, дата публикации
- Введение: 2–3 предложения о чём видео
- Ключевые идеи: буллетами, 3–7 пунктов
- Детальный разбор: тематические подразделы
- Выводы: резюме и основные мысли

### 8.5 Чат

#### Доступ

- Доступен без авторизации
- Без лимитов на количество вопросов (MVP)

#### Идентификация треда

- Для каждого видео создаётся (или переиспользуется) отдельный чат-тред
- Идентификатор треда хранится на клиенте в HTTP-only cookie (недоступен JS)
- Cookie позволяет продолжать диалог:
  - после перезагрузки страницы
  - при открытии новой вкладки с тем же видео
  - до истечения срока жизни cookie

#### История

- История сообщений хранится в Mastra AI (в рамках треда)
- При открытии страницы видео фронт запрашивает историю с бэкенда
- Бэкенд читает cookie, определяет тред и возвращает историю

#### Срок жизни

- Cookie имеет TTL (например, 30 дней) и автоматически продлевается при активности (sliding expiration)

#### Ограничения ввода

- Максимальная длина вопроса: 500 символов
- Если превышено: поле не позволяет вводить больше, показывается счётчик символов

#### Ответы ассистента

- Текст ответа на вопрос (на русском языке)
- 1–3 цитаты (если релевантные фрагменты найдены)

Формат цитаты:

```
"...фрагмент текста из субтитров..."
— 12:30–13:15
```

Если цитаты не найдены:
- Ответ содержит дисклеймер: «Не удалось найти точный фрагмент в видео.»

#### Ошибки доставки

Если ответ не получен (ошибка сети, таймаут):
- Сообщение: «Не удалось получить ответ. Попробуйте ещё раз.»
- Кнопка «Повторить» рядом с неотправленным вопросом

#### Безопасность cookie (MVP)

- Cookie выставляется с флагами: `HttpOnly`, `Secure` (в проде), `SameSite=Lax`
- Cookie привязано к домену приложения
- Прямой доступ к истории треда возможен только через бэкенд (фронт не имеет доступа к идентификатору треда)

### 8.6 Список видео (главная)

Отображаемые поля:

- Превью (thumbnail)
- Название
- Канал
- Статус
- Дата добавления

Возможности:

- Поиск по названию/каналу
- Фильтр по статусу (мультивыбор)
- Фильтр по дате добавления
- Сортировка: по дате (по умолчанию), по названию
- Пагинация (server-side)

### 8.7 Админка: задачи

Страница `/admin/jobs`:

- Список задач обработки
- Отображаемые поля: видео, тип задачи, статус, время старта/окончания, текст ошибки
- Фильтры: статус, период
- Переход к видео по клику

## 9. Нефункциональные требования

### Производительность

- Время обработки: <3 минут для видео до 60 минут
- Время загрузки страницы (FCP): <2 секунд

### Доступность

- Проект работает при наличии docker-compose

### Адаптивность

- Корректное отображение на мобильных устройствах (responsive design)

### Устойчивость

- Фоновые задачи ретраятся при сбое (до 3 попыток)

### Защита от спама

- Rate limit на добавление URL (5 в минуту с IP)
- Rate limit на чат по IP/видео, чтобы ограничить LLM-расходы

### Доступность интерфейса

- Базовая семантика: alt-тексты для изображений, aria-labels для интерактивных элементов
